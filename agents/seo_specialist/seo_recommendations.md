# SEO-рекомендации для блога NeoFamily

> Дата: 2026-03-17
> Блог: neofamily.ru/blog (WordPress)
> Конкурент: umschool.net/journal
> Плагин: Yoast SEO

---

## Аудит текущего состояния

### neofamily.ru — что есть сейчас
- Title оптимизирован: "NeoFamily: Онлайн-школа подготовки к ЕГЭ-2025 по всем предметам на 80+ баллов"
- Meta-description **отсутствует** — критическая проблема
- OG-теги **отсутствуют** — при шаринге в соцсетях отображается мусор
- Schema.org / JSON-LD **не внедрены**
- Breadcrumbs **отсутствуют**
- Блог на `/blog` **не существует** (404)
- Адаптивный дизайн есть, breakpoints настроены
- GTM интегрирован

### umschool.net/journal — что делает конкурент
- URL-иерархия: `/journal/[категория]/[slug-статьи]/`
- Schema.org внедрен: BlogPosting, Organization, WebSite, BreadcrumbList
- Авторские страницы: `/journal/author/[username]/` с аватарами
- 9 категорий с дропдаун-описаниями
- Слабости: нет H1 на главной журнала, отсутствуют alt у изображений, неполные breadcrumbs, короткий title

## 1. Структура URL

### Принципы
- Только латиница, lowercase, дефисы
- Trailing slash обязателен (настроить в Yoast)
- Максимальная глубина — 3 уровня

### Иерархия

```
neofamily.ru/blog/                              — главная блога
neofamily.ru/blog/aktualnoe/                    — категория
neofamily.ru/blog/ege-i-postuplenie/            — категория
neofamily.ru/blog/biologiya/                    — категория
neofamily.ru/blog/himiya/                       — категория
neofamily.ru/blog/russkij-yazyk/                — категория
neofamily.ru/blog/ege-i-postuplenie/kak-sdat-ege-na-90-ballov/  — статья
neofamily.ru/blog/author/ivan-petrov/           — страница автора
neofamily.ru/blog/tag/ege-2026/                 — тег (noindex, follow)
```

### Настройки WordPress / Yoast
- Permalinks: `/blog/%category%/%postname%/`
- Категории: использовать slug без `/category/` префикса (убрать через Yoast)
- Теги: `noindex, follow` — не засоряют индекс, но передают вес
- Пагинация: `rel="next"` / `rel="prev"`, страницы 2+ — `noindex, follow`

### Что нельзя делать
- Кириллица в URL (umschool это делает — их слабость)
- Даты в URL (`/2026/03/`) — статьи стареют визуально
- Параметры `?page=2` вместо `/page/2/`

## 2. Иерархия заголовков

### Главная блога (`/blog/`)

```
H1: "Блог NeoFamily — готовься к экзаменам с экспертами" (1 шт., уникальный)
  H2: "Актуальное" (секция)
    H3: [Заголовок статьи-карточки]
  H2: "ЕГЭ и поступление"
    H3: [Заголовок статьи-карточки]
  H2: "Популярные материалы" (сайдбар/блок)
    H3: [Заголовок статьи]
```

### Страница категории (`/blog/biologiya/`)

```
H1: "Биология — статьи и материалы для подготовки" (1 шт.)
  H2: [Заголовок статьи-карточки]
  H2: "Смотрите также" (блок перелинковки)
    H3: [Ссылки на смежные категории]
```

### Страница статьи (`/blog/biologiya/stroenie-kletki/`)

```
H1: "Строение клетки: что нужно знать для ЕГЭ по биологии" (1 шт.)
  H2: [Основные разделы статьи — 3-7 штук]
    H3: [Подразделы]
  H2: "Итоги" или "Запомни главное"
  H2: "Читай также" (блок перелинковки)
```

### Страница автора (`/blog/author/ivan-petrov/`)

```
H1: "Иван Петров — преподаватель биологии" (1 шт.)
  H2: "Об авторе"
  H2: "Статьи автора"
    H3: [Заголовки статей]
```

### Правила
- **Ровно один H1** на каждой странице
- H1 содержит основной ключ страницы
- Не пропускать уровни (H1 -> H3 без H2 — ошибка)
- В дизайне: H1 визуально самый крупный, но CSS может менять размер — семантика важнее визуала
- Название сайта в шапке — НЕ заголовок, а `<a>` или `<span>`

## 3. Внутренняя перелинковка

### Обязательные блоки в дизайне

#### a) Блок "Читай также" (в статье)
- Расположение: после 2-3 абзаца и в конце статьи
- 3 карточки, подобранные по категории/тегам
- Дизайн: горизонтальные мини-карточки (изображение + заголовок)

#### b) Блок "Популярное в категории" (сайдбар или после статьи)
- 5 самых читаемых статей текущей категории
- Дизайн: список со счётчиком просмотров

#### c) Хлебные крошки (breadcrumbs)
- На каждой странице кроме главной блога
- Формат: `Блог > Биология > Строение клетки`
- Каждый элемент — кликабельная ссылка
- Микроразметка BreadcrumbList (подробнее в разделе 6)

#### d) Теги внизу статьи
- Облако тегов: `#ЕГЭ-2026` `#биология` `#клетка`
- Кликабельные, ведут на `/blog/tag/[slug]/`

#### e) Блок автора
- Внизу статьи: аватар + имя + "Все статьи автора"
- Ссылка на `/blog/author/[slug]/`

#### f) Навигация по категориям
- Фиксированное меню блога с категориями
- Активная категория выделена визуально

### Стратегия перелинковки
- Каждая статья ссылается минимум на 2-3 другие статьи внутри текста (контекстные ссылки)
- Категории "Биология" и "Химия" перелинкованы (смежные предметы)
- "ЕГЭ и поступление" ссылается на предметные категории
- "Актуальное" — агрегатор, ссылается на всё
- Якорные тексты — содержательные, не "читать тут"

## 4. Meta-данные в дизайне

### Title (тег `<title>`)

| Тип страницы | Шаблон | Пример |
|---|---|---|
| Главная блога | Блог NeoFamily — подготовка к ЕГЭ и ОГЭ | — |
| Категория | {Категория} — статьи для подготовки \| NeoFamily | Биология — статьи для подготовки \| NeoFamily |
| Статья | {Заголовок статьи} \| NeoFamily | Строение клетки для ЕГЭ \| NeoFamily |
| Автор | {Имя} — преподаватель {предмет} \| NeoFamily | Иван Петров — преподаватель биологии \| NeoFamily |

- Длина: 50-60 символов
- Настраивается через Yoast SEO шаблоны

### Meta Description

| Тип страницы | Шаблон |
|---|---|
| Главная блога | Статьи, разборы заданий и советы для подготовки к ЕГЭ и ОГЭ от преподавателей NeoFamily. Биология, химия, русский язык и актуальные новости. |
| Категория | Всё о {категория}: разборы, теория, практика и советы от экспертов NeoFamily. Готовься к экзаменам эффективно. |
| Статья | {Первые 150 символов статьи или ручной текст в Yoast} |

- Длина: 140-160 символов
- Yoast SEO: шаблон `%%excerpt%%` для статей, ручная правка для важных страниц

### Open Graph (OG)

Обязательные теги для каждой страницы:
```html
<meta property="og:title" content="{Title страницы}">
<meta property="og:description" content="{Meta description}">
<meta property="og:image" content="{URL изображения 1200x630}">
<meta property="og:url" content="{Канонический URL}">
<meta property="og:type" content="article"> <!-- для статей -->
<meta property="og:site_name" content="NeoFamily">
<meta property="og:locale" content="ru_RU">
```

### Что учесть в дизайне
- **OG-изображение**: макет 1200x630px для каждой статьи. В дизайне предусмотреть шаблон: логотип NeoFamily + заголовок + фон категории
- **Twitter Card**: `summary_large_image` — аналогичное изображение
- Yoast генерирует OG автоматически, но изображения нужно загружать руками или задать дефолтное

## 5. Core Web Vitals — требования к дизайну

### LCP (Largest Contentful Paint) < 2.5s

**Что делать в дизайне:**
- Hero-изображение главной блога: максимум 200KB, формат WebP/AVIF
- Размеры изображений для карточек: 400x300px (превью), не больше
- Главное изображение статьи: 800x450px, `fetchpriority="high"`
- Шрифты: максимум 2 начертания, `font-display: swap`, preload основного шрифта
- Не использовать карусели/слайдеры в верхней части страницы

### CLS (Cumulative Layout Shift) < 0.1

**Что делать в дизайне:**
- Все изображения: обязательно указывать `width` и `height` в HTML
- Aspect ratio для карточек: фиксированный (например, 4:3)
- Шрифты: fallback с аналогичными метриками (`size-adjust` в @font-face)
- Рекламные блоки (если будут): зарезервировать место через `min-height`
- Lazy-loaded контент: placeholder с фиксированными размерами
- Шапка: фиксированная высота, не меняется при загрузке

### INP (Interaction to Next Paint) < 200ms

**Что делать в дизайне:**
- Меню категорий: CSS-only, без JS-анимаций при открытии
- Кнопки "Читать далее": нативные `<a>`, не JS-обработчики
- Фильтры/поиск: debounce 300ms
- Избегать тяжёлых анимаций при скролле

### Изображения — чеклист

| Элемент | Формат | Размер | Lazy load | Aspect ratio |
|---|---|---|---|---|
| Hero блога | WebP | 1200x600, <150KB | Нет (above fold) | 2:1 |
| Карточка статьи | WebP | 400x300, <50KB | Да | 4:3 |
| Изображение в статье | WebP | 800x450, <100KB | Да | 16:9 |
| OG-изображение | PNG/JPG | 1200x630, <300KB | — | ~1.91:1 |
| Аватар автора | WebP | 150x150, <20KB | Да | 1:1 |

### Обязательные атрибуты
```html
<img src="photo.webp" alt="Описание" width="800" height="450" loading="lazy" decoding="async">
```
- `loading="lazy"` — все изображения ниже fold
- `decoding="async"` — не блокирует рендер
- `alt` — описательный, с ключевыми словами (но без спама)

## 6. Structured Data (Schema.org / JSON-LD)

### Какие schema нужны

#### a) Organization (на всех страницах, в `<head>`)
```json
{
  "@context": "https://schema.org",
  "@type": "EducationalOrganization",
  "name": "NeoFamily",
  "url": "https://neofamily.ru",
  "logo": "https://neofamily.ru/logo.png",
  "sameAs": [
    "https://vk.com/neofamily",
    "https://t.me/neofamily"
  ]
}
```

#### b) WebSite + SearchAction (главная сайта)
```json
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "NeoFamily",
  "url": "https://neofamily.ru",
  "potentialAction": {
    "@type": "SearchAction",
    "target": "https://neofamily.ru/blog/?s={search_term_string}",
    "query-input": "required name=search_term_string"
  }
}
```

#### c) Blog (главная блога)
```json
{
  "@context": "https://schema.org",
  "@type": "Blog",
  "name": "Блог NeoFamily",
  "url": "https://neofamily.ru/blog/",
  "description": "Статьи и материалы для подготовки к ЕГЭ и ОГЭ"
}
```

#### d) BlogPosting (каждая статья) — ключевой тип
```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "Строение клетки: что нужно знать для ЕГЭ",
  "description": "Разбор строения клетки для ЕГЭ по биологии...",
  "image": "https://neofamily.ru/blog/wp-content/uploads/stroenie-kletki.webp",
  "datePublished": "2026-03-10",
  "dateModified": "2026-03-15",
  "author": {
    "@type": "Person",
    "name": "Иван Петров",
    "url": "https://neofamily.ru/blog/author/ivan-petrov/",
    "jobTitle": "Преподаватель биологии",
    "image": "https://neofamily.ru/blog/wp-content/uploads/authors/ivan-petrov.webp"
  },
  "publisher": {
    "@type": "EducationalOrganization",
    "name": "NeoFamily",
    "logo": {
      "@type": "ImageObject",
      "url": "https://neofamily.ru/logo.png"
    }
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://neofamily.ru/blog/biologiya/stroenie-kletki/"
  }
}
```

#### e) BreadcrumbList (все страницы кроме главной)
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {"@type": "ListItem", "position": 1, "name": "Блог", "item": "https://neofamily.ru/blog/"},
    {"@type": "ListItem", "position": 2, "name": "Биология", "item": "https://neofamily.ru/blog/biologiya/"},
    {"@type": "ListItem", "position": 3, "name": "Строение клетки"}
  ]
}
```

#### f) FAQPage (для статей с вопросами-ответами)
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Сколько баллов нужно для поступления на бюджет?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Проходной балл зависит от вуза..."
      }
    }
  ]
}
```

### Реализация в WordPress
- Yoast SEO генерирует: Organization, WebSite, BreadcrumbList, Article (BlogPosting)
- Вручную добавить: FAQPage (через ACF + кастомный блок или плагин)
- Проверка: Google Rich Results Test после каждой публикации

## 7. E-E-A-T — дизайн авторских страниц

### Почему это критически важно
Google оценивает Experience, Expertise, Authoritativeness, Trustworthiness. Для образовательного контента (YMYL-смежная тема — влияет на будущее школьников) E-E-A-T особенно важен.

### Страница автора — обязательные элементы

```
┌─────────────────────────────────────────────┐
│  [Фото 150x150]  Иван Петров               │
│                   Преподаватель биологии     │
│                   Стаж: 8 лет               │
│                   Выпускники: 500+           │
│                   Средний балл учеников: 85  │
├─────────────────────────────────────────────┤
│  Об авторе:                                 │
│  Окончил МГУ, биологический факультет.      │
│  Автор курса подготовки к ЕГЭ по биологии   │
│  в NeoFamily. Эксперт ЕГЭ с 2018 года.     │
├─────────────────────────────────────────────┤
│  Ссылки: [VK] [Telegram] [YouTube]          │
├─────────────────────────────────────────────┤
│  Статьи автора (23)                         │
│  ┌──────┐ ┌──────┐ ┌──────┐                │
│  │Card 1│ │Card 2│ │Card 3│                │
│  └──────┘ └──────┘ └──────┘                │
└─────────────────────────────────────────────┘
```

### Элементы E-E-A-T в дизайне статьи

#### Шапка статьи
- Дата публикации и дата обновления (видимые, не только в meta)
- "Проверено экспертом" — бейдж, если статья прошла ревью
- Время чтения: "7 мин"
- Категория — кликабельная

#### Блок автора внизу статьи
- Фото + имя + должность + краткое bio (2 строки)
- Кнопка "Все статьи автора"
- Соцсети автора

#### Источники и ссылки
- Блок "Источники" в конце статьи
- Ссылки на ФИПИ, официальные документы, учебники
- Повышает Trust

#### Рецензент (опционально, для ключевых статей)
- "Рецензент: [Имя], кандидат наук"
- Schema: `"reviewedBy": {"@type": "Person", ...}`

### Микроразметка автора
Yoast SEO генерирует `author` в BlogPosting. Дополнительно:
- Заполнить профиль автора в WordPress полностью (bio, соцсети)
- Подключить Gravatar или загрузить фото через User Profile
- В Yoast: включить "Author archives" с noindex=false

## Чеклист для дизайнера

### На каждом макете предусмотреть:
- [ ] Место для breadcrumbs (под шапкой, над H1)
- [ ] Один H1 — визуально главный заголовок
- [ ] Блок автора (аватар + имя + должность)
- [ ] Даты: публикации и обновления
- [ ] Время чтения
- [ ] Блок "Читай также" (3 карточки)
- [ ] Блок тегов
- [ ] Кнопки шаринга (генерируют OG-превью)
- [ ] Фиксированные размеры для всех изображений
- [ ] Мобильная версия каждого элемента

### Для разработчика (передать вместе с макетами):
- [ ] Yoast SEO: настроить шаблоны title/description для всех типов страниц
- [ ] JSON-LD: внедрить все 6 типов schema из раздела 6
- [ ] Изображения: WebP с fallback, `width`/`height`, lazy load
- [ ] Breadcrumbs: через Yoast (включить в настройках)
- [ ] Canonical URL: автоматически через Yoast
- [ ] Robots: теги `noindex` для пагинации и тегов
- [ ] Sitemap: Yoast генерирует, проверить включение /blog/ разделов
- [ ] Alt-теги: обязательное поле при загрузке изображений в WP

## Преимущества над umschool.net/journal

Что мы делаем лучше конкурента:
1. **Латинские URL** — они используют кириллицу, мы нет
2. **H1 на каждой странице** — у них отсутствует на главной журнала
3. **Alt-теги обязательны** — у них пропущены
4. **Полные breadcrumbs** — у них только 1 уровень
5. **E-E-A-T авторские страницы** — полноценные профили экспертов с квалификацией
6. **Даты обновления видимые** — повышают доверие и свежесть
7. **FAQ schema** — дополнительный rich snippet в выдаче
