## Оценка по направлениям

### 1. CTA-элементы и конверсия

**Что хорошо:**

- 2 CTA-баннера в статье размещены правильно: после вступления (когда читатель уже вовлечён) и перед заключением (когда задумывается "что дальше"). Это оптимальные позиции.
- CTA-виджет в sidebar — постоянно в поле зрения при чтении на десктопе. Градиент #6A5C90 -> #E9585A визуально выделяет его среди контента.
- Hero-баннер на главной с CTA-кнопкой — правильная точка входа в воронку.
- Кнопка "Войти" в header контурная (#E9585A) — не конкурирует с основными CTA, но привлекает внимание зарегистрированных.
- Блок обратной связи на каждой странице — дополнительная точка захвата лидов.
- Min-height 180px для CTA-баннеров — нет CLS, баннеры не "прыгают". Это хорошо и для UX, и для Core Web Vitals.

**Замечания:**

- Кнопки шеринга размещены ТОЛЬКО после рейтинга (внизу статьи). Для длинных лонгридов (2000-3000 слов) это поздно. Нужен **sticky шеринг сбоку** на десктопе или дублирование под заголовком. Человек, который дочитал до середины и хочет поделиться, не будет скроллить вниз.
- В sidebar нет блока email-подписки. Sidebar — это prime real estate для retention. Рекомендую добавить компактный виджет подписки под "Популярное".

---

### 2. Навигация для ЦА (школьники 15-18)

**Что хорошо:**

- Двухуровневый header с категориями — школьник видит все предметы сразу, не надо кликать в меню. Это снижает когнитивную нагрузку.
- Цветовое кодирование категорий (Биология = зелёный, Химия = голубой) — создаёт визуальные якоря, ускоряет навигацию при повторных визитах.
- Accent bar под заголовком секции в цвете категории — мгновенная визуальная привязка.
- Фильтры по предметам на странице категории — горизонтальные теги-кнопки интуитивны, привычны для ЦА (аналог TikTok/YouTube фильтров).
- Breadcrumbs на 3 уровня — школьник всегда понимает, где он находится.
- Sticky header — категории доступны в любой момент без скролла вверх.
- Поиск с автодополнением — быстрый доступ к конкретному запросу.

**Замечания:**

- Блок "Смотрите также" на странице категории — только текстовые ссылки на другие категории. Для школьников 15-18 лучше работают **визуальные карточки категорий** (иконка/цвет + название). Текстовые ссылки могут быть проигнорированы.
- На mobile категории в бургер-меню — это нормально, но стоит упомянуть **горизонтальный скролл категорий** как альтернативу бургеру. Школьники привыкли к горизонтальному свайпу (Stories, TikTok). Бургер = лишний клик = потеря.

---

### 3. Вовлекающие элементы

**Что хорошо:**

- Прогресс-бар чтения — мотивирует дочитать ("осталось 20%"), снижает отказы. Для лонгридов по ЕГЭ это критично.
- Рейтинг "Была ли статья полезна?" — простой лайк/дизлайк, минимальный порог вовлечения. Правильный выбор для ЦА.
- Hover-эффект карточек (поднятие + scale изображения) — создаёт ощущение интерактивности.
- Пустое состояние поиска с иллюстрацией и популярными запросами — удерживает пользователя вместо тупика.
- Анимация стрелки "Читать ещё" при hover — мелочь, но добавляет живость.
- Бейдж "Проверено экспертом" — повышает доверие и дочитываемость.

**Замечания:**

- **Нет комментариев.** Это было в моих рекомендациях. Комментарии = UGC = дополнительный контент для SEO + вовлечение + возвратные визиты. Рекомендую добавить хотя бы в P1. Даже простая форма "Задай вопрос по теме" работает лучше, чем ничего.
- **Нет таймера до ЕГЭ.** Для ЦА это мощный мотиватор ("До ЕГЭ осталось 67 дней"). Виджет в sidebar занял бы минимум места, но создавал бы urgency. Рекомендую в P1.

---

### 4. Что лишнего / отвлекает от контента

**Что хорошо:**

- Главная компактная — 4 карточки на категорию вместо бесконечной ленты. Правильное решение.
- Hero — одно изображение, не карусель. Нет отвлекающей анимации.
- CTA-баннеры нативно вписаны (градиент/цвет категории, border-radius 24px) — не вызывают баннерную слепоту.

**Замечания:**

- **Блок обратной связи на КАЖДОЙ странице** (включая статью) — перегруз. На странице статьи уже есть: рейтинг + шеринг + теги + блок автора + источники + "Читай также" + обратная связь + footer. Это 7 блоков после основного контента. Рекомендую **убрать блок обратной связи со страницы статьи** — он дублирует функцию CTA-баннеров и формы в footer. Оставить на главной и категориях.
- Inline ToC + sidebar ToC = дублирование на десктопе. Пользователь видит оглавление дважды. Рекомендую на десктопе inline ToC делать **свёрнутым по умолчанию** (только заголовок "Содержание" с toggle), а в sidebar показывать развёрнутый. На mobile — inline развёрнутый.

---

### 5. Карточки статей — привлекательность и CTR

**Что хорошо:**

- Все ключевые элементы для CTR на месте: изображение 4:3, категория с цветовой меткой, время чтения, заголовок (2 строки), описание (2 строки), автор с аватаром, дата.
- Line-clamp на 2 строки для заголовка и описания — нет "оборванных" текстов, карточки одинаковой высоты.
- Hover-эффект (подъём + тень + scale картинки) — интерактивность, приглашение к клику.
- Метка категории uppercase 12px с цветным фоном — мгновенная идентификация предмета.
- Border-radius 20px — современно, дружелюбно, соответствует духу бренда для молодой аудитории.

**Замечания:**

- **Нет индикатора "Новое" или "Обновлено"** на карточках. Для блога, который будет обновлять контент (даты обновления видны в статье), полезно показывать это уже на карточке. Бейдж "Обновлено" повышает CTR на 10-15% для вернувшихся пользователей.
- **Описание карточки 14px** — мелковато. Учитывая, что ЦА читает с телефона, 14px на mobile может быть нечитаемым. В брифе указано "Минимум на mobile: 16px body". Рекомендую поднять описание до 15-16px или убедиться, что на mobile масштабируется.

## Итоговая оценка

### Что хорошо (кратко)

- CTA-баннеры расположены в правильных точках воронки (после вступления, перед заключением, sidebar)
- Навигация интуитивная: двухуровневый header + цветовое кодирование + фильтры — школьник найдёт нужное за 1-2 клика
- Карточки статей содержат все элементы для CTR: картинка, категория, время, заголовок, автор
- Прогресс-бар + рейтинг + hover-эффекты создают достаточную интерактивность
- Компактная главная без перегруза — правильный подход
- E-E-A-T проработан: авторские страницы, бейдж эксперта, источники, даты обновления
- Чередование белого/серого фона для секций — визуальный ритм, страница не сливается
- OG-шаблон для соцсетей — повысит CTR при шеринге

### Замечания (что доработать)

1. **Sticky шеринг сбоку (desktop)** или дублирование под заголовком — для лонгридов шеринг только внизу = потеря расшариваний
2. **Email-подписка в sidebar** — добавить компактный виджет под "Популярное" для retention
3. **Комментарии** — добавить в концепцию хотя бы как P1 (после блока "Читай также")
4. **Таймер до ЕГЭ в sidebar** — мощный мотиватор для ЦА, добавить в P1
5. **Убрать блок обратной связи со страницы статьи** — перегруз (7 блоков после контента)
6. **Inline ToC свёрнутый на desktop** — избежать дублирования с sidebar ToC
7. **Бейдж "Обновлено" на карточках** — повышает CTR для вернувшихся пользователей
8. **Описание карточки поднять до 15-16px** — 14px мелковато для ЦА на mobile
9. **"Смотрите также" на странице категории** — заменить текстовые ссылки на визуальные мини-карточки с цветом категории
10. **Mobile: горизонтальный скролл категорий** вместо (или в дополнение к) бургер-меню — привычный паттерн для ЦА 15-18

### Критичные проблемы

Критичных проблем, блокирующих запуск, нет. Концепция покрывает ~80% маркетинговых рекомендаций и все P0-требования из брифа. Замечания выше — это доработки уровня P1, которые повысят конверсию и вовлечение, но не блокируют создание макетов.
