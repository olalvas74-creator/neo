# WordPress Blog NeoFamily — Чеклист технических требований

Домен: `https://neofamily.ru/blog`
CMS: WordPress
Обязательный плагин: Yoast SEO

---

## 1. Страницы и URL-структура

### 1.1 Иерархия URL

- [ ] `/blog/` — главная страница блога
- [ ] `/blog/{категория}/` — страница категории (напр. `/blog/biologiya/`)
- [ ] `/blog/{категория}/{подкатегория}/` — страница подкатегории (напр. `/blog/ege/matematika/`)
- [ ] `/blog/{категория}/{подкатегория}/{slug-статьи}/` — страница статьи
- [ ] `/blog/author/{author-slug}/` — авторская страница

### 1.2 Необходимые страницы

- [ ] **Главная блога** — секции по категориям, по 4 карточки в каждой, кнопка "Читать ещё"
- [ ] **Страница категории** — список статей категории с пагинацией
- [ ] **Страница подкатегории** — список статей подкатегории с пагинацией
- [ ] **Страница статьи** — полный контент статьи со всеми элементами (ToC, рейтинг, шеринг и т.д.)
- [ ] **Страница автора** — фото, должность, опыт, описание, сертификаты, 4 последние публикации
- [ ] **Страница поиска** — результаты поиска по блогу

### 1.3 Категории контента (из дизайна)

- [ ] Актуальное
- [ ] ЕГЭ и поступление
- [ ] Биология
- [ ] Химия
- [ ] Русский язык
- [ ] (расширяемый список — новые категории через админку)

### 1.4 Пермалинки и роутинг

- [ ] Настроить permalink structure для вложенной иерархии `/blog/%category%/%postname%/`
- [ ] Обеспечить корректный rewrite rules для подкатегорий
- [ ] Убедиться, что нет конфликтов с основным сайтом (блог в подпапке или subpath)
- [ ] 301-редиректы при изменении slug-ов

## 2. Компоненты

### 2.1 Header (единый на всех страницах)

- [ ] Логотип + текст "NeoFamily. Блог" — выравнивание влево
- [ ] Навигационное меню справа: "Банк заданий", "Магазин", "Войти"
- [ ] Ссылки меню ведут на соответствующие разделы основного сайта
- [ ] Фильтры категорий блога — горизонтальная панель под основной навигацией
- [ ] Фильтры кликабельны, ведут на страницы категорий
- [ ] Активная категория визуально выделена
- [ ] Sticky header (фиксация при скролле) — уточнить с дизайном
- [ ] Адаптивное меню (бургер на мобильных)

### 2.2 Footer (единый на всех страницах)

- [ ] Единый дизайн на всех страницах блога
- [ ] Навигационные ссылки
- [ ] Контактная информация
- [ ] Копирайт
- [ ] Ссылки на соцсети (если есть)

### 2.3 Блок обратной связи (перед футером)

- [ ] Располагается на каждой странице между контентом и футером
- [ ] Форма: имя, email/телефон, сообщение
- [ ] Валидация на клиенте и сервере
- [ ] Защита от спама (reCAPTCHA или honeypot)
- [ ] Отправка на email администратора или в CRM
- [ ] Сообщение об успешной отправке

### 2.4 Карточка статьи (превью)

- [ ] Обложка статьи (thumbnail)
- [ ] Заголовок (ссылка на статью)
- [ ] Категория (ссылка на категорию)
- [ ] Дата публикации
- [ ] Автор (ссылка на страницу автора)
- [ ] Hover-эффект
- [ ] Корректное отображение при длинных заголовках (truncate или line-clamp)

### 2.5 Блок вовлечения

- [ ] Промо-блок на главной блога (CTA, подписка, или информационный баннер)
- [ ] Управляемый из админки (контент, ссылка, видимость)

### 2.6 Breadcrumbs

- [ ] Присутствуют на всех страницах (кроме главной блога)
- [ ] Иерархия: Главная → Категория → Подкатегория → Статья
- [ ] Интеграция с Yoast SEO breadcrumbs
- [ ] JSON-LD разметка breadcrumbs

### 2.7 Пагинация

- [ ] 15–20 статей на страницу (настраивается в админке)
- [ ] Навигация: номера страниц + prev/next
- [ ] Работает на страницах категорий и подкатегорий
- [ ] SEO-friendly URL: `/blog/biologiya/page/2/`
- [ ] `rel="prev"` / `rel="next"` в `<head>`

## 3. Функциональность

### 3.1 Оглавление (Table of Contents)

- [ ] Автогенерация из заголовков H2 внутри статьи
- [ ] Якорные ссылки для каждого H2
- [ ] Кликабельная навигация (scroll to anchor)
- [ ] Возможность скрыть/показать (toggle)
- [ ] Позиционирование: перед первым H2 или в sidebar
- [ ] Корректная работа при отсутствии H2

### 3.2 Система рейтинга

- [ ] Рейтинг статьи (звёзды, 1–5 или лайк/дизлайк)
- [ ] Защита от повторного голосования (cookie / IP / auth)
- [ ] Отображение среднего рейтинга и количества оценок
- [ ] Данные рейтинга в JSON-LD (aggregateRating)
- [ ] AJAX-отправка без перезагрузки страницы

### 3.3 Кнопки шеринга

- [ ] Социальные сети: VK, Telegram, Одноклассники, WhatsApp
- [ ] Кнопка "Копировать ссылку"
- [ ] Позиция: сбоку статьи (sticky) или в конце статьи
- [ ] Не загружать сторонние SDK — использовать share URL
- [ ] Open Graph теги для корректного превью при шеринге

### 3.4 Шорткоды и блоки

- [ ] **Баннер-шорткод** — настраиваемый промо-баннер внутри статьи
  - [ ] Параметры: изображение, заголовок, текст, ссылка, стиль
  - [ ] Gutenberg-блок + fallback-шорткод
- [ ] **Цитата** — стилизованный блок цитаты (blockquote с кастомным дизайном)
- [ ] **Полноширинное изображение** — выход за пределы контентной колонки (alignfull)

### 3.5 Превью автора (в статье)

- [ ] Блок внизу статьи перед комментариями/шерингом
- [ ] Фото автора
- [ ] Имя (ссылка на авторскую страницу)
- [ ] Краткое описание / должность
- [ ] Ссылка "Все статьи автора"

### 3.6 Авторская страница

- [ ] Фото автора (крупное)
- [ ] Имя, должность, опыт работы (стаж)
- [ ] Развёрнутое описание / биография
- [ ] Список сертификатов и достижений (E-E-A-T сигналы)
- [ ] 4 последние публикации автора (карточки)
- [ ] Ссылка на полный список публикаций

### 3.7 Метаданные статьи

- [ ] Дата публикации
- [ ] Дата последнего обновления
- [ ] Автор
- [ ] Категория / подкатегория
- [ ] Время чтения (расчёт по количеству слов)
- [ ] Обложка статьи (featured image)

## 4. SEO и Structured Data

### 4.1 Yoast SEO

- [ ] Установка и настройка Yoast SEO Premium (или бесплатной версии)
- [ ] Настройка шаблонов title и description для всех типов страниц
- [ ] XML Sitemap — включён, корректная структура
- [ ] Breadcrumbs через Yoast
- [ ] Canonical URL на всех страницах
- [ ] Настройка robots.txt
- [ ] Подключение Google Search Console
- [ ] Подключение Яндекс.Вебмастер

### 4.2 JSON-LD Schema

- [ ] **Article** (или BlogPosting) на страницах статей:
  - [ ] headline, datePublished, dateModified
  - [ ] author (type: Person, name, url)
  - [ ] publisher (type: Organization, name, logo)
  - [ ] image (обложка)
  - [ ] description
  - [ ] aggregateRating (если рейтинг включён)
  - [ ] mainEntityOfPage
- [ ] **BreadcrumbList** на всех страницах с breadcrumbs
- [ ] **Person** на авторских страницах (имя, должность, описание, фото)
- [ ] **WebSite** с SearchAction на главной блога
- [ ] **Organization** — общая разметка сайта

### 4.3 Валидация

- [ ] Все страницы проходят Google Rich Results Test
- [ ] Нет ошибок structured data в Search Console
- [ ] Нет ошибок в Яндекс.Вебмастер — Микроразметка

### 4.4 Meta-теги и Open Graph

- [ ] `<title>` — уникальный для каждой страницы
- [ ] `<meta name="description">` — уникальный для каждой страницы
- [ ] Open Graph теги: og:title, og:description, og:image, og:url, og:type
- [ ] Twitter Card теги (если нужно)
- [ ] `<link rel="canonical">`
- [ ] hreflang (если будет мультиязычность — сейчас не требуется)

### 4.5 Семантическая вёрстка

- [ ] `<header>` — шапка сайта
- [ ] `<nav>` — навигация, breadcrumbs
- [ ] `<main>` — основной контент страницы
- [ ] `<article>` — каждая статья
- [ ] `<section>` — логические секции
- [ ] `<aside>` — sidebar (если используется)
- [ ] `<footer>` — подвал
- [ ] Корректная иерархия заголовков H1–H6 (один H1 на страницу)

## 5. Адаптивность и производительность

### 5.1 Адаптивность

- [ ] Брейкпоинты: mobile (< 768px), tablet (768–1024px), desktop (> 1024px)
- [ ] Header: бургер-меню на мобильных, полное меню на десктопе
- [ ] Карточки статей: 1 колонка (mobile), 2 колонки (tablet), 4 колонки (desktop)
- [ ] Полноширинные изображения корректно масштабируются
- [ ] Таблицы внутри статей — горизонтальный скролл на mobile
- [ ] Шрифты читаемы на всех устройствах (min 16px body)
- [ ] Touch-friendly элементы (min 44x44px для кликабельных зон)
- [ ] Тестирование на реальных устройствах (iOS Safari, Android Chrome)

### 5.2 Производительность

- [ ] Core Web Vitals в зелёной зоне (LCP < 2.5s, FID < 100ms, CLS < 0.1)
- [ ] Lazy loading для изображений (`loading="lazy"`)
- [ ] Оптимизация изображений: WebP формат, srcset для разных размеров
- [ ] Минификация CSS и JS
- [ ] Кэширование: серверное (object cache, page cache) + браузерное (cache headers)
- [ ] CDN для статики (если применимо)
- [ ] Критический CSS inline, остальной — async
- [ ] Минимум сторонних скриптов
- [ ] Gzip / Brotli сжатие на сервере
- [ ] Отложенная загрузка JS (`defer` / `async`)
- [ ] Предзагрузка шрифтов (`<link rel="preload">`)
- [ ] Database query optimization (минимум запросов на страницу)

### 5.3 Доступность (a11y)

- [ ] `alt` атрибуты у всех изображений
- [ ] Контрастность текста (WCAG AA, ratio >= 4.5:1)
- [ ] Навигация с клавиатуры (tab, enter)
- [ ] ARIA-атрибуты где необходимо
- [ ] Skip-to-content ссылка
