# Бриф для дизайнера: Блог NeoFamily

> Дата: 2026-03-17
> Составлено на основе: дизайн-токены, WP-требования, профиль компании, анализ конкурента (Умскул), маркетинг-рекомендации, SEO-рекомендации

---

## 1. О компании

- **Бренд:** NeoFamily
- **Позиционирование:** Онлайн-школа подготовки к ЕГЭ на 80+ баллов
- **Домен:** neofamily.ru (основной), neofam.online (мерч, амбассадоры)
- **Направления:** курсы, банк заданий ЕГЭ, справочник (smart-directory) по 11 предметам
- **Ключевые предметы:** биология, химия, русский язык, математика
- **Продукты:** годовые / полугодовые / экспресс-курсы (2 490 -- 8 490 руб.), банк заданий с решениями, справочные материалы
- **Сейчас блог на /blog не существует** (404). Статьи размещены в формате smart-directory по предметам. Создаём блог с нуля.

---

## 2. Цель проекта

Создать блог на WordPress по адресу `neofamily.ru/blog`, который:

1. **Привлекает органический трафик** по запросам, связанным с ЕГЭ, предметами и поступлением
2. **Конвертирует читателей в учеников** через CTA-баннеры и формы записи на курсы
3. **Строит экспертный образ** NeoFamily (E-E-A-T) через авторские страницы и качественный контент
4. **Превосходит конкурента** (Умскул Журнал) по UX, навигации, SEO-оснащению и вовлечению

## 3. Целевая аудитория

| Сегмент | Описание | Приоритет |
|---------|----------|-----------|
| Ученики 10-11 классов | Готовятся к ЕГЭ, ищут разборы заданий, теорию, лайфхаки | Основной |
| Ученики 9 классов | Потенциально ОГЭ (расширение) | Второстепенный |

- Привыкли к неформальному тону, обращению на "ты"
- Ценят конкретику: баллы, сроки, пошаговые инструкции

## 4. Страницы для дизайна

### 4.1. Главная блога (`/blog/`)

**Назначение:** витрина контента, точка входа из поиска и с основного сайта.

**Содержимое:**
- Header (общий)
- H1: "Блог NeoFamily -- готовься к экзаменам с экспертами"
- Hero-баннер -- промо текущей акции / курса с CTA-кнопкой (управляется из админки)
- Секция "Актуальное" -- 4 карточки последних статей
- Секции по категориям (ЕГЭ и поступление, Биология, Химия, Русский язык) -- по 4 карточки в каждой + кнопка "Читать ещё"
- Блок вовлечения -- промо-баннер или форма подписки (управляется из админки)
- Блок обратной связи (форма: имя, email/телефон, сообщение)
- Footer (общий)

**Сетка карточек:** 1 колонка (mobile), 2 колонки (tablet), 4 колонки (desktop).

**Не перегружать:** у конкурента главная слишком длинная -- делаем компактнее.

---

### 4.2. Страница категории (`/blog/{категория}/`)

**Назначение:** список статей по категории с возможностью фильтрации.

**Содержимое:**
- Breadcrumbs: Блог > Категория
- H1: "{Категория} -- статьи и материалы для подготовки"
- Описание категории (1-2 предложения, управляется из админки)
- Фильтры по предметам (внутри ЕГЭ) -- горизонтальные теги-кнопки (**наше преимущество над конкурентом**)
- Карточки статей с пагинацией (15-20 на страницу)
- Блок "Смотрите также" -- ссылки на смежные категории
- Блок обратной связи
- Footer

**Пагинация:** номера страниц + prev/next. URL: `/blog/biologiya/page/2/`

---

### 4.3. Страница подкатегории (`/blog/{категория}/{подкатегория}/`)

**Аналогична странице категории**, но с breadcrumbs: Блог > Категория > Подкатегория.

---

### 4.4. Страница статьи (`/blog/{категория}/{slug}/`)

**Назначение:** основной контент, главная страница для SEO-трафика.

**Содержимое (подробно в разделе 8):**
- Breadcrumbs: Блог > Категория > Статья
- Прогресс-бар чтения (sticky top)
- H1: заголовок статьи
- Мета-блок: автор (аватар + имя-ссылка) | дата публикации | дата обновления | время чтения | категория
- Бейдж "Проверено экспертом" (опционально)
- Оглавление (Table of Contents) -- сворачиваемое
- Тело статьи с H2/H3, списками, изображениями, цитатами, баннерами
- Sidebar (desktop): sticky-оглавление, CTA-виджет, "Популярное в категории"
- Блок рейтинга: "Была ли статья полезна?" (звёзды или лайк/дизлайк)
- Кнопки шеринга: VK, Telegram, WhatsApp, копировать ссылку
- Теги статьи (облако)
- Блок автора (фото + имя + должность + био 2 строки + "Все статьи автора")
- Блок "Источники" (ссылки на ФИПИ, учебники)
- Блок "Читай также" -- 3 карточки похожих статей
- Блок обратной связи
- Footer

---

### 4.5. Страница автора (`/blog/author/{slug}/`)

**Назначение:** E-E-A-T сигнал, доверие к контенту.

**Содержимое:**
- Breadcrumbs: Блог > Автор
- Крупное фото автора (150x150)
- H1: "{Имя} -- преподаватель {предмет}"
- Должность, стаж, количество выпускников, средний балл учеников
- Развёрнутая биография (2-3 абзаца)
- Список сертификатов и достижений
- Ссылки на соцсети автора (VK, Telegram, YouTube)
- H2: "Статьи автора" -- 4 последние публикации (карточки) + ссылка "Все статьи"
- Footer

---

### 4.6. Страница поиска (`/blog/?s=...`)

- Поле поиска с автодополнением
- Количество найденных результатов
- Карточки статей (стандартные)
- Пагинация
- Блок "Популярные запросы" (если результатов мало)

- Trailing slash обязателен
- Без дат в URL
- Без кириллицы (преимущество над конкурентом)
- Максимальная глубина -- 3 уровня

- Навигационные ссылки (разделы блога + основной сайт)
- Контактная информация
- Ссылки на соцсети (VK, Telegram, YouTube)
- Копирайт
- Единый дизайн на всех страницах

### 7.3. Блок обратной связи (перед footer, все страницы)

- Форма: имя, email/телефон, сообщение
- CTA-заголовок: "Остались вопросы?"
- Кнопка отправки (primary, #E9585A)
- Сообщение об успешной отправке

### 7.4. Карточка статьи (превью)

```
+----------------------------------+
|  [Изображение 4:3, WebP]        |
|                                  |
|  [Категория]  [5 мин чтения]    |
```

- Изображение: 400x300px, aspect ratio 4:3 фиксированный, lazy load

- Дата публикации
- Hover-эффект (поднятие тени или масштаб)
- Border-radius: 20-24px

- На всех страницах кроме главной блога
- Формат: Блог > Категория > Подкатегория > Статья
- Каждый элемент кликабельный
- Расположение: под header, над H1
- Шрифт: Inter 14px, цвет #5C5964
- Интеграция с Yoast SEO breadcrumbs (JSON-LD BreadcrumbList)

### 7.6. Пагинация

- Номера страниц + стрелки prev/next
- Активная страница выделена (primary цвет)
- 15-20 статей на страницу
- Стиль: кнопки с border-radius 8px

### 7.7. Sidebar (desktop, на странице статьи)

- Sticky при скролле
- Оглавление статьи (дублирует ToC, следует за пользователем)
- CTA-виджет: "Запишись на курс" (ссылка на магазин NeoFamily)

### 7.8. Hero-баннер (главная блога)

- Промо текущей акции / курса
- Яркое изображение + заголовок + CTA-кнопка

- Максимум 200KB, WebP
- Не карусель (одно изображение) -- для скорости загрузки

## 8. Контентные блоки статьи

Порядок элементов сверху вниз:

### 8.1. Breadcrumbs
Блог > Биология > Строение клетки

### 8.2. Прогресс-бар чтения
- Sticky top (поверх header или под ним)
- Тонкая линия, цвет primary (#E9585A)
- Заполняется по мере скролла

### 8.3. Заголовок и мета

```
H1: Строение клетки: что нужно знать для ЕГЭ по биологии

[Аватар] Иван Петров  |  10 марта 2026  |  Обновлено: 15 марта  |  7 мин  |  Биология
[Бейдж: Проверено экспертом] (опционально)
```

### 8.4. Оглавление (Table of Contents)

- Автогенерация из H2 заголовков статьи
- Располагается перед первым H2
- Сворачиваемое (toggle "Содержание")
- Якорные ссылки, smooth scroll
- Визуально: серый фон (#F9F8FB), border-radius 16px, padding 20px
- При отсутствии H2 -- не показывается

### 8.5. Тело статьи

Поддерживаемые блоки контента (Gutenberg):

#### Текст
- Параграфы: Nunito 18px, line-height 1.6, цвет #333333
- H2: Comfortaa 32px, якорная ссылка (иконка # при hover)
- H3: Comfortaa 26px
- Списки: маркированные и нумерованные, с отступом

#### Цитата (blockquote)
- Левая граница 4px, цвет primary (#E9585A)
- Фон: #F9F8FB
- Шрифт: Nunito 20px italic
- Border-radius: 0 16px 16px 0
- Источник цитаты мелким шрифтом

#### Изображение в статье
- Max-width 100% контентной колонки (720px)
- Подпись (caption): Inter 14px, цвет #5C5964
- Lazy load, WebP, 800x450px max
- Обязательно `width`, `height`, `alt`

#### Полноширинное изображение (alignfull)
- Выходит за пределы контентной колонки на всю ширину
- Для инфографик, таблиц, диаграмм

#### CTA-баннер (шорткод / Gutenberg-блок)
- 2 штуки на статью: после вступления + перед заключением
- Содержимое: изображение, заголовок, текст, кнопка со ссылкой
- Стиль: яркий фон (градиент или цвет категории), border-radius 24px
- Управляется из админки (параметры: изображение, текст, ссылка, стиль)
- Нативно вписан в контент (не баннерная слепота)

#### Таблица
- Responsive: горизонтальный скролл на mobile
- Зебра-полосы (чередующиеся строки)
- Border-radius: 12px на обёртке

#### Код / формула
- Моноширинный блок с фоном #F0F0F0
- Border-radius: 8px

### 8.6. Блок рейтинга

```
Была ли статья полезна?
[Лайк] 45  [Дизлайк] 3
```

- Простая оценка лайк/дизлайк или 5 звёзд
- AJAX, без перезагрузки
- Отображение количества оценок
- JSON-LD aggregateRating

### 8.7. Кнопки шеринга

```
Поделиться: [VK] [Telegram] [WhatsApp] [Копировать ссылку]
```

- Позиция: после рейтинга или сбоку (sticky на desktop)
- Без загрузки сторонних SDK -- share URL
- Иконки: round buttons, 44x44px touch target
- При копировании -- тултип "Скопировано!"

### 8.8. Теги статьи

```
#ЕГЭ-2026  #биология  #клетка  #цитология
```

- Кликабельные, ведут на `/blog/tag/{slug}/`
- Стиль: tag-компоненты из UI Kit, border-radius 4-8px
- Цвет фона: #EFEDF2

### 8.9. Блок автора

```
+-----------------------------------------------------+
| [Фото 64px]  Иван Петров                            |
|              Преподаватель биологии, стаж 8 лет      |
|              Эксперт ЕГЭ, автор курса в NeoFamily   |

```

- Фон: #F9F8FB, border-radius 20px
- Фото: 64x64, круглое
- Имя -- ссылка на `/blog/author/{slug}/`
- Краткое bio: 2 строки max

### 8.10. Блок "Источники"

- Список: нумерованный
- Ссылки на ФИПИ, учебники, официальные документы
- Шрифт: Inter 14px
- Повышает Trust (E-E-A-T)

### 8.11. Блок "Читай также"

- 3 карточки похожих статей (горизонтальные мини-карточки: изображение + заголовок)
- Подбор по категории/тегам
- H2: "Читай также"

### 8.12. Блок "Популярное в категории" (inline, mobile)

- На mobile заменяет sidebar
- 5 статей списком (заголовок + дата)
- Показывается после "Читай также"

## 9. SEO-требования к дизайну

### 9.1. Заголовки
- Ровно один H1 на каждой странице, визуально самый крупный
- Иерархия H1 > H2 > H3 без пропусков
- Логотип в шапке -- НЕ заголовок (использовать `<a>` или `<span>`)

### 9.2. Семантическая вёрстка
- `<header>` -- шапка
- `<nav>` -- навигация, breadcrumbs
- `<main>` -- основной контент
- `<article>` -- каждая статья
- `<section>` -- логические секции
- `<aside>` -- sidebar
- `<footer>` -- подвал

### 9.3. Изображения -- обязательные атрибуты

| Элемент | Формат | Размер | Lazy load | Aspect ratio |
|---------|--------|--------|-----------|--------------|
| Hero блога | WebP | 1200x600, <150KB | Нет (above fold) | 2:1 |
| Карточка статьи | WebP | 400x300, <50KB | Да | 4:3 |
| Изображение в статье | WebP | 800x450, <100KB | Да | 16:9 |
| OG-изображение | PNG/JPG | 1200x630, <300KB | -- | ~1.91:1 |
| Аватар автора | WebP | 150x150, <20KB | Да | 1:1 |

Все изображения: обязательно `width`, `height`, `alt`, `loading="lazy"` (кроме above fold), `decoding="async"`.

### 9.4. OG-изображение (шаблон)
Предусмотреть дизайн шаблона 1200x630px:
- Логотип NeoFamily
- Заголовок статьи
- Фоновый цвет категории
- Используется при шаринге в соцсетях

### 9.5. Core Web Vitals

**LCP < 2.5s:**
- Никаких каруселей/слайдеров в верхней части
- Hero максимум 200KB
- Шрифты: preload, swap

**CLS < 0.1:**
- Фиксированные размеры для всех изображений (width/height в HTML)
- Фиксированный aspect ratio карточек
- Header фиксированной высоты
- Зарезервированное место для CTA-баннеров (min-height)

**INP < 200ms:**
- Меню категорий: CSS-only, без JS-анимаций
- Кнопки "Читать далее": нативные `<a>`, не JS
- Избегать тяжёлых анимаций при скролле

### 9.6. Контрастность
- WCAG AA: ratio >= 4.5:1 для текста
- Проверить все комбинации цвет текста / фон

### 9.7. Место для мета-данных
В дизайне предусмотреть видимое расположение:
- Даты публикации и обновления (не только в meta-тегах)
- Время чтения
- Категория

## 10. Технические ограничения

### 10.1. WordPress -- обязательно

- CMS: WordPress + Gutenberg
- Плагин SEO: Yoast SEO (breadcrumbs, schema, sitemap)
- Блог живёт в подпапке `/blog/` основного сайта
- Permalink structure: `/blog/%category%/%postname%/`

### 10.2. Что нельзя делать

- Кириллица в URL
- Даты в URL (`/2026/03/`)
- Карусели/слайдеры в hero-зоне (медленные)
- Загрузка сторонних SDK для шеринга (только share URL)
- Больше одного H1 на страницу
- Пропуск уровней заголовков (H1 > H3 без H2)
- Квадратные изображения карточек (использовать 4:3)
- JS-only навигация (категории должны быть обычными ссылками)
- Pop-up при уходе со страницы (раздражает ЦА)
- Параметры `?page=2` вместо `/page/2/`

### 10.3. Обязательные технические элементы

- Адаптивность: mobile (<768px), tablet (768-1024px), desktop (>1024px)
- Touch target: минимум 44x44px для кликабельных зон
- Skip-to-content ссылка (a11y)
- Навигация с клавиатуры (tab, enter)
- ARIA-атрибуты где необходимо
- `rel="prev"` / `rel="next"` для пагинации
- Canonical URL на всех страницах
- reCAPTCHA или honeypot на формах

### 10.4. Брейкпоинты

| Брейкпоинт | Ширина | Карточки | Sidebar | Меню |
|------------|--------|----------|---------|------|
| Mobile | <768px | 1 колонка | Скрыт | Бургер |
| Tablet | 768-1024px | 2 колонки | Скрыт | Бургер или сокращённое |
| Desktop | >1024px | 4 колонки | Виден | Полное |

## 11. Референсы

### 11.1. Что берём у конкурента (Умскул Журнал)

| Элемент | Как у конкурента | Как берём |
|---------|-----------------|-----------|
| Лонгриды 2000-3000 слов | Работает для SEO | Берём как есть |
| Оглавление в начале статьи | Есть, работает | Берём + делаем sticky в sidebar |
| Иерархия H2/H3 | Чёткая | Берём как есть |
| Тон на "ты" | Неформальный | Берём как есть |
| Авторство на каждой статье | Имя + аватар | Берём + расширяем (должность, стаж) |
| 2 CTA-баннера внутри статьи | Нативные | Берём как есть |
| Schema.org разметка | BlogPosting, BreadcrumbList | Берём + добавляем FAQPage, полные breadcrumbs |
| Блок "Читай по теме" | 3 статьи в конце | Берём + добавляем inline после 2-3 абзаца |

### 11.2. Что делаем по-своему (наши преимущества)

| Наше решение | Проблема конкурента |
|-------------|---------------------|
| Фильтры по предметам внутри категорий | У них нет, неудобно искать нужный предмет |
| Поиск по блогу с автодополнением | У них поиск скрыт или отсутствует |
| Sidebar со sticky-оглавлением и CTA | У них нет sidebar |
| Время чтения на карточках | У них нет -- влияет на CTR |
| Кнопки шеринга VK/Telegram/WhatsApp | У них только Telegram-бот |
| Рейтинг статьи ("Полезна ли?") | У них нет |
| Полноценные авторские страницы (E-E-A-T) | У них минимальные профили |
| Латинские URL | У них кириллица |
| H1 на главной блога | У них отсутствует |
| Обязательные alt у изображений | У них пропущены |
| Полные breadcrumbs (3 уровня) | У них только 1 уровень |
| Видимые даты обновления | У них только дата публикации |
| Прогресс-бар чтения | У них нет |
| Email-подписка с лид-магнитом | У них нет retention-механик |
| Компактная главная (не перегруженная) | У них слишком длинная страница |
